# UVSP 协议解析器 — 代码框架分析

## 项目概述

这是一个 **UVSP（UDP Video Stream Protocol）视频流协议解析器**，功能是从 UDP 网络包中接收视频数据，解析 UVSP 协议头，将数据包按序重组成完整帧，并通过 `ffplay` 实时显示视频画面。项目使用 C++11 编写，基于 POSIX 线程(pthread)实现并发处理。

---

## 一、整体架构

```
┌───────────────────────────────────────────────────────────────┐
│                         main.cpp                              │
│   (入口: 初始化参数、设置回调、启动/停止解析器)                    │
└──────────┬────────────────────────────────────────────────────┘
           │

┌───────────────────────────────────────────────────────────────┐
│                     UvspParser (uvsp_parser.h/.cpp)           │
│                                                               │
│  ┌──────────────────┐   ┌──────────────────┐                  │
│  │   recv_thread    │   │   parse_thread   │                  │
│  │  (UDP 接收线程)  │──▶│  (协议解析线程)   │                  │
│  └────────┬─────────┘   └────────┬─────────┘                  │
│           │                      │                            │
│           ▼                      ▼                            │
│    ┌──────────────┐      ┌──────────────┐                     │
│    │   RingQueue  │      │ FrameAssembler│                    │
│    │  (无锁环形队列)│     │  (帧重组器)    │                    │
│    └──────────────┘      └──────┬───────┘                     │
│                                 │                             │
│                                 ▼                             │
│                          ┌──────────────┐                     │
│                          │ display_thread│                    │
│                          │ (ffplay 显示)  │                    │
│                          └──────────────┘                     │
└───────────────────────────────────────────────────────────────┘
```

### 线程模型（3 个线程）

| 线程 | 函数 | 职责 |
|------|------|------|
| **接收线程** | `recv_thread` / `recv_loop` | 从 UDP 端口(17762) 接收原始数据包，放入无锁队列 |
| **解析线程** | `parse_thread` / `parse_loop` | 从队列取出数据，解析协议头，进行包级回调 + 帧重组 |
| **显示线程** | `display_thread`（每个流一个） | 将完整帧送入 `ffplay` 进行实时显示 |

## 二、输入部分（UDP 数据接收）

### 2.1 位置
- **文件**: `src/uvsp_full_parser.cpp`
- **函数**: `UvspParser::recv_loop()`

### 2.2 流程

```cpp
// src/uvsp_full_parser.cpp — recv_loop()

// 1. 创建 UDP socket，绑定 17762 端口
sock_ = socket(AF_INET, SOCK_DGRAM, 0);
bind(sock_, (sockaddr*)&addr, sizeof(addr));

// 2. 循环接收数据（最大 2048 字节/包）
while (running_) {
    int len = recv(sock_, buf, 2048, 0);

    // 3. 封装为 Packet 结构体，推入无锁队列
    Packet p;
    p.data = new uint8_t[len];
    memcpy((void*)p.data, buf, len);
    p.len = len;

    queue_.push(p);   // ← 生产-消费者模式
}
```

### 2.3 关键数据结构

| 结构/变量 | 用途 |
|-----------|------|
| `Packet` | 原始数据包，包含 `data`(数据指针) 和 `len`(长度) |
| `RingQueue` | 无锁环形队列（容量 4096），线程间传递 `Packet` |
| `sock_` / `port_`(17762) | UDP socket 和监听端口 |

### 2.4 说明
- 采用 **生产-消费者模型**：接收线程是生产者，解析线程是消费者
- 使用自实现的无锁环形队列 `RingQueue`（基于 `head_`/`tail_` 原子操作），避免锁竞争
- 每个包动态分配内存，解析线程释放

## 三、解析部分（协议解析）

### 3.1 位置
- **文件**: `src/uvsp_full_parser.cpp`
- **函数**: `UvspParser::parse_loop()`

### 3.2 协议结构

```
┌──────────────────────────────────────────────────┐
│                  UDP Payload                      │
├────────────────┬─────────────────┬───────────────┤
│  BaseHeader    │ CommonTransport │  ext_tvl[]    │
│  (8 bytes)     │   (8 bytes)     │  (可选, 扩展)  │
├────────────────┴─────────────────┴───────────────┤
│                    Payload (数据体)                │
└──────────────────────────────────────────────────┘
```

#### BaseHeader (基础头, 8字节)
| 字段 | 大小 | 说明 |
|------|------|------|
| `magic` | 2B | 魔数: `0x5654`(Full) / `0x5653`(Lite) |
| `flags` | 1B | 标志位: SOF(0x02), EOF(0x01), GAP(0x04), RST(0x08) |
| `profile` | 1B | 配置文件 |
| `source_id` | 2B | 源 ID（流标识） |
| `type` | 2B | 数据类型 |

#### CommonTransport (公共传输头, 8字节)
| 字段 | 大小 | 说明 |
|------|------|------|
| `seq_num` | 2B | 序列号（用于排序和丢包检测） |
| `timestamp` | 4B | 时间戳 |
| `ext_count` | 1B | 扩展字段数量 |
| `reserved` | 1B | 保留 |

### 3.3 解析流程

```cpp
// src/uvsp_full_parser.cpp — parse_loop()

// 1. 从队列取包
queue_.pop(p);

// 2. 验证魔数 magic
BaseHeader* base = (BaseHeader*)p.data;
if (magic != MAGIC_FULL && magic != MAGIC_LITE) → 丢弃

// 3. 提取基础字段
uint16_t id   = ntohs(base->source_id);   // 流 ID
uint8_t flags = base->flags;              // 标志位
uint16_t seq  = ntohs(t->seq_num);        // 序列号
uint32_t ts   = ntohl(t->timestamp);      // 时间戳

// 4. 计算 payload 偏移
//    头大小 = sizeof(BaseHeader) + sizeof(CommonTransport) + sizeof(ext_tvl) * ext_count
//    payload 从 header_size 开始，长度 = p.len - header_size

// 5. 包级回调（用户可注册）
pkt_cb_(info, pkt_user_);

// 6. 解析 TLV 扩展段（如果 ext_count > 0）
//    从中提取 CHUNK_OFFSET（分片偏移）和 TOTAL_SIZE（帧总大小）
FrameAssembler::parse_tlvs(ext_data, ext_count, &has_offset, &chunk_offset, &total_size);

// 7. 送入帧重组器（带 TLV 信息，内部自动选择拼接路径）
assembler_.input_ext(id, seq, flags, timestamp, payload, payload_len,
                     has_offset, chunk_offset, total_size);
```

### 3.4 TLV 扩展解析（新功能）

新增 `parse_tlvs()` 静态函数，用于解析协议文档定义的 TLV 格式扩展字段：

| TLV 类型 | 常量 | Value 长度 | 说明 |
|----------|------|-----------|------|
| 0x01 | TLV_FRAME_ID | 4B | 帧 ID |
| 0x02 | TLV_CHUNK_OFFSET | 4B | 分片在帧内的字节偏移 |
| 0x03 | TLV_CHUNK_LENGTH | 4B | 有效载荷长度 |
| 0x04 | TLV_TOTAL_SIZE | 4B | 帧总大小 |
| 0x05 | TLV_PIXFMT | 4B | 像素格式 |
| 0x06 | TLV_WIDTH | 4B | 宽度 |
| 0x07 | TLV_HEIGHT | 4B | 高度 |

TLV 结构定义：

```cpp
struct ExtTlvHeader {
    uint8_t  type;       // 扩展类型
    uint8_t  len;        // Value 长度
    uint16_t reserved;   // 4B 对齐
    // uint8_t value[];  // 变长数据
};
```

当前 `parse_tlvs()` 只提取 `CHUNK_OFFSET` 和 `TOTAL_SIZE`，其余 TLV 类型预留供后续扩展。

### 3.5 序列号统计

`update_seq()` 函数维护每个流 ID 的 **64-bit 位图滑动窗口**，统计：
- **丢包数** (`lost`): 序列号跳跃超过窗口大小时计数
- **乱序数** (`reorder`): 旧序列号首次出现
- **重复数** (`duplicate`): 旧序列号重复出现

### 3.6 说明
- 使用 `ntohs`/`ntohl` 进行网络字节序到主机字节序的转换
- 支持 **Full** (0x5654) 和 **Lite** (0x5653) 两种协议格式
- 包级回调在 `main.cpp` 的 `on_packet()` 中实现，用于打印和日志记录

## 四、输出显示部分（帧重组 + 实时显示）

### 4.1 位置
- **文件**: `src/uvsp_full_parser.cpp`
- **类**: `FrameAssembler`

### 4.2 帧重组流程（双路径拼接）

`FrameAssembler` 现在支持**两条拼接路径**，由 `input_ext()` 统一入口，内部自动选择：

```
input_ext() 收到包
     ↓
SOF 时检查 has_offset
     ↓
has_offset = true ──────────→ 路径 B：偏移拼接
     │                          使用 offset_map<offset, data>
     │                          按 CHUNK_OFFSET 精确定位到 buf 位置
     ↓
has_offset = false  ─────────→ 路径 A：顺序拼接（原逻辑）
                                使用 pkt_map<seq, data>
                                按 seq 连续性依次拼接
```

#### 路径 A：顺序拼接（原逻辑，兼容旧协议）

```cpp
// input() — 完全不动
// 依赖 pkt_map（按 seq 排序）+ expected_seq 连续性

f.pkt_map[seq] = vector<uint8_t>(payload, payload + len);

while (true) {
    auto it = f.pkt_map.find(f.expected_seq);
    if (it == f.pkt_map.end()) break;

    memcpy(f.buf + f.received, data, copy_len);
    f.received += copy_len;
    f.expected_seq++;
}

if (f.received >= FRAME_SIZE) { ... 帧完成 ... }
```

#### 路径 B：偏移拼接（新功能，基于 TLV）

```cpp
// input_ext() — 当 has_offset=true 时走此路径
// 依赖 offset_map + expected_offset 连续拼接

// 重复 offset 跳过，不重复累加
if (f.offset_map.find(chunk_offset) != f.offset_map.end())
    return;
f.offset_map[chunk_offset] = vector<uint8_t>(payload, payload + len);

// 从 expected_offset 开始连续拼，拼到断为止
while (true) {
    auto it = f.offset_map.find(f.expected_offset);
    if (it == f.offset_map.end()) break;

    memcpy(f.buf + f.expected_offset, data.data(), data.size());
    f.filled_size += data.size();
    f.expected_offset += data.size();
    f.offset_map.erase(it);
}

if (f.filled_size >= f.total_size) { ... 帧完成 ... }
```

两种路径互不干扰，同一流的不同包可以各自走对应路径。

### 4.3 显示流程

```cpp
// display_thread() — ffplay 管道路径

// 1. 构造 ffplay 命令（通过管道写入）
snprintf(cmd, sizeof(cmd),
    "ffplay -fflags nobuffer -flags low_delay "
    "-f rawvideo -pixel_format %s -video_size %dx%d "
    "-framerate %d -x 640 -y 512 -window_title cam_%d -",
    g_cfg.pixel_format, g_cfg.width, g_cfg.height,
    g_cfg.fps, f->id);

// 2. 打开 ffplay 进程
FILE* player = popen(cmd, "w");

// 3. 循环从帧队列取数据，写入 ffplay 的 stdin
while (f->display_running) {
    auto frame = f->frame_queue.front();
    f->frame_queue.pop();
    fwrite(frame.data(), 1, frame.size(), player);  // 写入原始视频数据
    fflush(player);
}
```

### 4.4 关键数据结构

| 结构/变量 | 用途 |
|-----------|------|
| `FrameBuf` | 每个流 ID 的帧缓冲区状态（帧缓存、乱序 map、偏移 map、显示队列等） |
| `pkt_map` | `std::map<uint16_t, vector<uint8_t>>` — 按序列号排序的乱序包缓冲（路径A） |
| `offset_map` | `std::map<uint32_t, vector<uint8_t>>` — 按偏移量排序的分片缓冲（路径B） |
| `filled_size` | 路径B已填充字节数 |
| `expected_offset` | 路径B期望的下一个偏移（类似路径A的 expected_seq） |
| `total_size` | 路径B帧总大小（从 TLV_TOTAL_SIZE 读取） |
| `use_offset_mode` | 当前帧是否走偏移拼接 |
| `frame_queue` | `queue<vector<uint8_t>>` — 已完成帧的队列，限流 3 帧 |
| `frames_` | `unordered_map<uint16_t, FrameBuf>` — 多流管理 |

### 4.5 说明
- 使用 **`ffplay` 子进程** 通过管道接收原始视频数据，实现低延迟显示
- 显示队列限流 **3 帧**，超出时丢弃旧帧，保证实时性
- 每个视频流 ID 独立拥有一个 `FrameBuf` 和对应的显示线程

## 五、主入口（main.cpp）

### 5.1 位置
- **文件**: `src/main.cpp`

### 5.2 流程

```cpp
int main() {
    // 1. 信号处理（优雅退出）
    signal(SIGINT, signal_handler);
    signal(SIGTERM, signal_handler);

    // 2. 创建解析器
    UvspParser parser;

    // 3. 命令行参数 → 视频配置
    //    raw60  → 1920x1536 60fps bayer_bggr8
    //    raw30  → 1920x1536 30fps bayer_bggr8
    //    yuv30  → 1920x1536 30fps yuyv422
    if (argc >= 2) { ... 解析参数 ... }

    // 4. 注册回调
    parser.set_packet_callback(on_packet, &parser);   // 包回调
    parser.set_frame_callback(on_frame, nullptr);     // 帧回调

    // 5. 启动解析器（创建接收线程 + 解析线程）
    parser.start();

    // 6. 主线程等待
    while (g_running) sleep(1);

    // 7. 停止解析器
    parser.stop();

    return 0;
}
```

### 5.3 视频配置

| 参数名 | 分辨率 | 帧率 | 像素格式 | 帧大小 |
|--------|--------|------|----------|--------|
| `raw60` | 1920×1536 | 60fps | bayer_bggr8 | 1920×1536 |
| `raw30` | 1920×1536 | 30fps | bayer_bggr8 | 1920×1536 |
| `yuv30` | 1920×1536 | 30fps | yuyv422 | 1920×1536×2 |

### 5.4 包回调 (`on_packet`)

打印包级信息 + 帧错误检测日志：

```cpp
// 打印包基本信息（仅 id=4/7 且含 SOF 标志的包）
cout << "[PKT] id=" << info.id << " seq=" << info.seq
     << " flags=0x" << hex << (int)info.flags << dec
     << " ts=" << info.timestamp
     << " payload_len=" << info.payload_len << endl;

// 检测 frame_err 变化 → 记录到 frame_error.log
if (s.frame_err > prev_err) {
    log_file << "[ERR] time=..." << " id=" << info.id
             << " frame_err_inc=" << (s.frame_err - prev_err)
             << " frame_lost=" << s.frame_lost << endl;
}
```

## 六、帧统计模块

### 6.1 位置
- **文件**: `src/uvsp_full_parser.cpp`
- **函数**: `update_frame()`

### 6.2 逻辑

```cpp
// 遇到 SOF → 结算上一帧
if (flags & FLAG_SOF) {
    if (s.frame_bytes > 0) {
        if (s.frame_bytes >= FRAME_SIZE)
            s.frame_ok++;           // 正常帧
        else {
            s.frame_err++;          // 异常帧（不完整）
            s.frame_lost += (FRAME_SIZE - s.frame_bytes);  // 丢失字节数
        }
    }
    s.frame_bytes = 0;  // 开始新帧
}

// 累加当前帧字节数
s.frame_bytes += payload;
```

### 6.3 统计指标

| 指标 | 含义 |
|------|------|
| `frame_ok` | 完整接收的帧数 |
| `frame_err` | 不完整帧数（丢数据导致） |
| `frame_lost` | 帧级别丢失的字节总数 |
| `ok_frame_size` | 最近成功帧的大小 |

## 九、总结

| 模块 | 文件 | 核心职能 |
|------|------|----------|
| **输入** | `uvsp_full_parser.cpp::recv_loop()` | UDP 网络数据接收，无锁队列缓冲 |
| **解析** | `uvsp_full_parser.cpp::parse_loop()` | UVSP 协议解析、TLV 扩展解析、序列号统计、帧错误检测 |
| **重组** | `FrameAssembler::input()` / `input_ext()` | 双路径拼接：路径A(顺序seq) + 路径B(偏移offset) |
| **显示** | `FrameAssembler::display_thread()` | 通过管道送 ffplay 实时显示视频 |
| **入口** | `main.cpp::main()` | 参数解析、回调注册、生命周期管理 |
## 当前实现状态（2026-06-18）

- UDP 接收端当前监听配置端口，真实调试中临时使用 `32896`；如设备改回目标端口，应同步改回 `17762`。
- 解析流程支持 `BaseHeader(8) + CommonTransport(8) + optional absolute_timestamp(8) + TLV + payload`。
- 绝对时间戳字段为 8B 大端 `uint64_t`，解析结果进入 `PacketInfo` / `FrameInfo`：
  - `has_absolute_timestamp`
  - `absolute_timestamp`
- 为兼容真实设备，`FLAG_TS=0x10` 不会被直接当成“必有 8B 字段”；解析器会校验 TLV 起点，自动选择旧格式或带绝对时间戳的新格式。
- 默认运行只输出帧级日志：
  `[FRAME] OK id=<id> ts=<u32> abs_ts_valid=<0/1> abs_ts=<u64> size=<bytes> pending_chunks=<n>`
- 包级日志和首包/首帧 dump 默认关闭，由 `DEBUG_PACKET_LOG`、`DEBUG_DUMP_FIRST_PACKET`、`DEBUG_DUMP_FIRST_FRAME` 控制。
