## 协议二进制结构（更新版 — 含 TLV 扩展）

```
偏移(字节) | 字段          | 大小(字节)   | 说明
-----------+---------------+-------------+---------------------------
 0         | magic         | 2           | 魔数: 0x5654(Full) / 0x5653(Lite)
 2         | flags         | 1           | 标志: SOF(0x02), EOF(0x01), GAP(0x04), RST(0x08)
 3         | profile       | 1           |
 4         | source_id     | 2           | 流ID（网络序）
 6         | type          | 2           |
-----------+---------------+-------------+---------------------------
 8         | seq_num       | 2           | 序列号（网络序）
10         | timestamp     | 4           | 时间戳（网络序）
14         | ext_count     | 1           | TLV 扩展项数量（决定 ext_tvl[] 大小）
15         | reserved      | 1           |
-----------+---------------+-------------+---------------------------
16         | ext_tvl[]     | 8×ext_count | 扩展字段，每项 8 字节（固定）
16+8*N     | payload       | 可变        | 视频数据
```

### 头大小计算公式

```cpp
header_size = sizeof(BaseHeader)          // 8
            + sizeof(CommonTransport)     // 8
            + sizeof(ext_tvl) * ext_count; // 8 × ext_count
payload = p.data + header_size;
payload_len = p.len - header_size;
```

## TLV 扩展项结构

每个 `ext_tvl` 项固定 8 字节，由 4 字节头 + 4 字节值组成：

```
字节 0-3     head (uint32)
字节 4-7     cmd  (uint32)
```

### 解析为结构化 TLV

实际 TLV 格式（UVSP-D-Full_Ext_VideoRaw_v1.0）：

```
┌──────┬──────┬──────────┬──────────┐
│ type │ len  │ reserved │  value   │
│ 1 B  │ 1 B  │  2 B     │ len 字节  │
└──────┴──────┴──────────┴──────────┘
       总大小 = 4 + len，4B 对齐
```

### TLV 类型常量

| 常量名             | 值  | 说明                     |
|--------------------|-----|--------------------------|
| TLV_FRAME_ID       | 0x01| 帧 ID                    |
| TLV_CHUNK_OFFSET   | 0x02| 分片偏移（uint32）       |
| TLV_CHUNK_LENGTH   | 0x03| 分片长度                  |
| TLV_TOTAL_SIZE     | 0x04| 帧总大小（uint32）       |
| TLV_PIXFMT         | 0x05| 像素格式                  |
| TLV_WIDTH          | 0x06| 图像宽度                  |
| TLV_HEIGHT         | 0x07| 图像高度                  |

### 解析流程（parse_tlvs）

```cpp
uint32_t offset = 0;
for (i = 0; i < ext_count; i++) {
    ExtTlvHeader* hdr = ext_data + offset;
    switch (hdr->type) {
        case TLV_CHUNK_OFFSET:
            chunk_offset = ntohl(*(uint32_t*)value);
            break;
        case TLV_TOTAL_SIZE:
            total_size = ntohl(*(uint32_t*)value);
            break;
    }
    offset += (sizeof(ExtTlvHeader) + hdr->len + 3) & ~3; // 4B 对齐
}
```

**注意**：实际验证中发现 `sizeof(ext_tvl) * ext_count`（8×N）正好等于 TLV 数据总大小，说明每个 TLV 恰好 8 字节（4B 头 + 4B 值），与协议规范一致。

## 双模拼帧逻辑（input_ext）

### 决策树

```
parse_tlvs 得到 has_offset, chunk_offset, total_size
                │
                ▼
         input_ext() 进入
                │
        ┌───────┴───────┐
        │               │
  chunk_offset==0   chunk_offset>0
        │               │
        ▼               ▼
   input()          偏移拼接
  (顺序拼接)        (新路径)
```

### 模式 1：顺序拼接（旧模式）

- 条件：`chunk_offset == 0` 或 `ext_count == 0`
- 行为：基于 seq 号按序拼接，`pkt_map[seq] → expected_seq → memcpy`
- 初始化：`received=0, pkt_map.clear()`

### 模式 2：偏移拼接（新模式）

- 条件：`chunk_offset > 0` 且 `ext_count > 0`
- 行为：基于 CHUNK_OFFSET 定位到 `buf` 的指定位置
- 初始化：`offset_map.clear(), filled_size=0, expected_offset=0`
- 拼接循环：检查 `offset_map[expected_offset]`，有则 memcpy，`expected_offset += data.size()`

### 帧完成判断

| 模式     | 完成条件              | 输出帧大小 |
|----------|----------------------|-----------|
| 顺序拼接 | `received >= FRAME_SIZE` | FRAME_SIZE（固定） |
| 偏移拼接 | `filled_size >= total_size` | filled_size（动态） |

## 关键数据结构（FrameBuf）

```cpp
struct FrameBuf {
    // === 公共 ===
    uint8_t* buf = nullptr;                    // 拼接缓冲区
    size_t received = 0;                       // 顺序模式：已接收字节数
    uint16_t id;                               // 流 ID
    uint16_t expected_seq = 0;                 // 顺序模式：期望的下一 seq
    uint32_t last_timestamp = 0;
    bool got_base = false;                     // 是否收到 SOF
    bool initialized = false;
    std::map<uint16_t, std::vector<uint8_t>> pkt_map;  // seq → 数据

    // === 偏移拼接模式专属 ===
    std::map<uint32_t, std::vector<uint8_t>> offset_map; // offset → 数据
    size_t filled_size = 0;                    // 已填充字节数
    uint32_t expected_offset = 0;              // 期望的下一 offset
    uint32_t total_size = 0;                   // 帧总大小（来自 TLV_TOTAL_SIZE）
    bool use_offset_mode = false;              // 是否启用偏移模式

    // === 显示 ===
    std::queue<std::vector<uint8_t>> frame_queue;  // 待显示帧队列
    pthread_mutex_t mutex;
    pthread_t display_tid;
    bool display_running = false;
};
```

## 实际数据验证

### TLV 解析结果（实测）

```
[TLV] cnt=1471485 id=4 seq=2546 ext_count=3 has_offset=1 chunk_offset=0 total_size=0
[TLV] cnt=2946052 id=1 seq=2093 ext_count=3 has_offset=1 chunk_offset=0 total_size=0
[TLV] cnt=4420610 id=8 seq=1348 ext_count=3 has_offset=1 chunk_offset=0 total_size=0
...
```

| 字段          | 值   | 说明                                     |
|---------------|------|-----------------------------------------|
| ext_count     | 3    | 每个包固定 3 个 TLV                     |
| has_offset    | true | 包含 TLV_CHUNK_OFFSET 类型              |
| chunk_offset  | 0    | 所有包 offset 均为 0（非递增）          |
| total_size    | 0    | 未读到 TLV_TOTAL_SIZE                   |

### 结论

当前发送端**协议实现不完整**：
- 有 TLV 格式（3 个扩展项，每项 8 字节）
- 有 CHUNK_OFFSET 字段，但值恒为 0
- 没有 TOTAL_SIZE
- 实际数据仍是顺序排列

因此当前代码在 `chunk_offset == 0` 时走顺序拼接，完全兼容。

## 当前实现状态（2026-06-18）

- 当前拼帧主链路为偏移拼帧：每个有效包必须解析到 `TLV_CHUNK_OFFSET`，按帧内 offset 写入缓冲区。
- `CommonTransport` 后支持可选 8B 绝对时间戳：`uint64_t absolute_timestamp`，大端，位置在 TLV 扩展段之前。
- 兼容真实设备现状：有些包设置 `FLAG_TS=0x10`，但实际没有插入 8B 绝对时间戳。解析器会同时校验两种 TLV 起始偏移：
  - 旧格式：`BaseHeader(8) + CommonTransport(8) + TLV`
  - 新格式：`BaseHeader(8) + CommonTransport(8) + absolute_timestamp(8) + TLV`
- `PacketInfo` / `FrameInfo` 已包含 `has_absolute_timestamp` 和 `absolute_timestamp`。
- 默认只打印帧级完成日志，例如：
  `[FRAME] OK id=1 ts=12345678 abs_ts_valid=1 abs_ts=123456789 size=2949120 pending_chunks=0`
- 详细包级日志和首包/首帧 dump 由 `src/uvsp_full_parser.cpp` 顶部开关控制：
  `DEBUG_PACKET_LOG`、`DEBUG_DUMP_FIRST_PACKET`、`DEBUG_DUMP_FIRST_FRAME`。
